### Arguments

When querying for `tests`, the following arguments are available. If you don't include a runId, the API defaults to the most recent run of the specified job:

<QueryArgsTable queryName="tests" />

Below we show some illustrative example queries and outline the schema (all possible fields you can query) of the tests object.

### Example query

The example query below finds all tests in this job and includes information about those tests.

```graphql
{
  tests(jobId: 123) {
    runId
    accountId
    projectId
    uniqueId
    name
    columnName
    state
  }
}
```
